## Refuted (with evidence)

- **"Variadic `--disallowedTools A B ÔÇŽ` may parse only the first tool"** (gemini, 85): live
  probe ÔÇö a cell with all tools space-separated returns `is_error: false` with a correct
  result; stray positionals would have broken the invocation. Commander-style `<tools...>`
  variadics consume until the next flag.
- **"`/tmp/` fragment will false-positive on legitimate model output"** (gemini): output
  fields are stripped before the scan ever runs ÔÇö the scanner sees only short structured
  fields, where any such fragment *is* a leak.
- **"Figures/README numbers may not reproduce"** (attack run by the blind voice): verified
  the opposite ÔÇö every README number reproduces from the committed ledger exactly.
- **"Pre-registration might be post-hoc"** (attack run by the blind voice): commit
  timestamps prove METHODOLOGY landed ~2h before the first sweep cell.
- **"Deny-list should be replaced by an allow-list CLI flag"** (codex): no such exclusive
  allow-list semantics exists in the current CLI (`--allowedTools` is an auto-approve list);
  catalog-verification backlogged instead (D6).
