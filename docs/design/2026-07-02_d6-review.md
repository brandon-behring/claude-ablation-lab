# D6 hardening PR — 3-voice adversarial review (2026-07-02)

**Subject:** `chore/d6-hardening` (task-scoped tool policy, live-verified tool catalog,
in-band mechanism capture, sanitizer keep-list inversion) — 3 commits, `main..chore/d6-hardening`
(`c83eba4` runtime, `a48b365` tests, `28da066` docs). **Voices:** codex (gpt-5.5 @xhigh) + Gemini
3.1 Pro via `adversarial_review.py --commit <sha>` (the single-pass diff, ~84KB, tripped the
big-diff guard — split by commit, never truncated), plus a blind fresh-context Claude subagent
over the full branch diff independently. Gemini timed out (>300s) on the two larger commits
(`c83eba4`, `a48b365`) — succeeded only on `28da066` (docs). Findings tool-grounded (live probes
+ direct execution) before acceptance, not accepted on confidence score alone.

**Tally: 19 findings pooled → 11 confirmed and fixed, 3 recorded as deferred (not blocking), 5
refuted with evidence.** One of the 11 fixes (excluding `StructuredOutput` from the hermetic
deny-list) closes a **live regression** this PR would otherwise have shipped: the new
`capture_mechanism=True` default, combined with T1's `json_schema`, would have silently broken
every T1 cell the moment `ablation run` was used against it. A second self-caught issue (not
from any of the three voices) closes a CI-portability bug: the new version gate would have
broken every non-dry-run `run`/`estimate` test on any machine — including GitHub Actions —
without a `claude` binary installed matching the pin.

## Deferred (recorded, not fixed — see `docs/design/2026-07-01_phase6-deferrals.md`)

- **D6.2**: a live `system/init` stream-json event's `tools` array is a second tool-enumeration
  source, disagreeing with the `--disallowedTools` probe on membership (omits `Grep`/`Glob`) in
  a way not yet understood. Not adopted as the primary catalog source until reconciled.
- **D6.3** (blind voice): `--tools <tools...>` is a genuine native restrictive allow-list flag,
  distinct from `--allowedTools` (which PR #11's review already correctly refuted as an
  auto-approve-only list, not a restriction). Worth a future spike, not a swap this late in an
  already-large, already-tested PR.
- `parse_stream_json` ignores `rate_limit_event`; a stream cut off before its terminal `result`
  could be classified `parse_fail`/`infra_error` instead of `rate_limited` (codex, confidence 48
  — the lowest-confidence finding pooled, and a pre-existing coarseness in the no-payload
  fallback path this PR doesn't worsen, not a new regression).

## Refuted (with evidence)

- **"`MultiEdit`/`NotebookRead`/`LS` are real tools missing from the catalog"** (codex, critical):
  wrong — `MultiEdit` and `NotebookRead` were directly probed and confirmed *fake* (identical
  "matches no known tool" warning as the deliberately-made-up control name); `LS` has no evidence
  of existing as a Claude Code tool name at all (file listing is `Glob`).
- **"`Read`/`Write` aren't real tool names — should be `View`/`Edit`/`Replace`"** (gemini,
  critical, confidence 95): wrong — directly contradicted by two independent live probes *and*
  this very session's own tool list.
- **"A T2 agent's `Bash`+`Write` access lets it tamper with the validator that grades it"**
  (gemini, critical, confidence 95): wrong — `ValidatorGrader` resolves the validator from the
  task's static `gold.toolkit_root` (the original repo), never from the cell's `cwd` (a separate
  worktree with independent file contents). Different paths on disk; writing to one doesn't
  touch the other. Recorded preemptively in the T2 runway doc since it's a reasonable question a
  future reader would independently ask.
- **"`KEEP_FIELDS` omitting `infra_repo` breaks distinguishing variants in plots"** (gemini,
  confidence 90): wrong — `variant` (unchanged, always present, unrelated to this PR) is what
  `analyze.py`'s SQL actually groups by everywhere; `infra_repo` was already excluded pre-PR
  and is redundant with it.
- **"`--verbose` might interleave debug text with stream-json, breaking per-line JSON parsing"**
  (gemini, confidence 75): wrong — directly refuted by the already-captured, already-clean real
  fixture (`--verbose` was on during capture) and by `parse_stream_json`'s demonstrated tolerance
  of stray non-JSON lines regardless.
